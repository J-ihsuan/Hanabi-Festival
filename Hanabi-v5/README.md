## Run the Application
**A. Setup & Build**
1. Open the Terminal and navigate to the project folder.

    ```bash
    cd Hanabi-v5
    ```

2. Install the required dependencies (TypeScript, Webpack, and Loaders):

    ```bash
    npm install
    ```
3. Use Webpack to bundle the modular TypeScript files into a single JavaScript file `dist/bundle.js`:

    ```bash
    npx webpack
    ```

**B. Launch Server**
1. Run the following command to start the local server:

    ```bash
    npm start 
    ```

**C. View Webpage**
* **On Computer**   

    Open a web browser (Chrome/Safari) and navigate to http://localhost:3000
* **On Mobile Phone**   

    Ensure the mobile device and computer are on the SAME Wi-Fi network.    
    Find the computer's Local IP address, and navigate to http://[YOUR_IP]:3000 on the mobile browser.
    ###### Example: http://192.168.1.16:3000

## Advanced TypeScript Features Used in This Assignment
**Interfaces (Contract Definition)**
* Implemented the `IEventData` interface to define a strict data contract for festival objects. This ensures that any class or object representing an event adheres to a specific structure, improving code reliability and type safety.

**ES Modules (Modular Architecture)**
* Utilized `export` and `import` syntax to de-construct the application into logical modules `eventModels.ts`, `eventRenderer.ts`, and `main.ts`. This makes the codebase more maintainable and scalable.

**Classes with Access Modifiers**
* Used the `implements` keyword to ensure classes correctly fulfill the requirements of defined interfaces.

**Webpack Bundling & Optimization**
* Configured Webpack as the module bundler to resolve dependencies between different TypeScript files. This transforms multiple development files into a single optimized `bundle.js`, reducing HTTP requests and simplifying deployment.



## Debugging TypeScript Code
Using **Source Maps** configured within the `tsconfig.json` and `webpack.config.js` files. Since browsers execute the bundled JavaScript rather than the original TypeScript, Source Maps act as a bridge that allows the browser's Developer Tools (Console/Sources) to map the execution back to the original `.ts` source files.
